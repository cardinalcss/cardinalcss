Getting started

Designers and developers who want to build well-organized, responsive web applications without losing their minds, should use Cardinal.

This framework omits aesthetic design decisions that can weigh down larger, more complicated CSS frameworks. It’s style-agnostic. The design and creativity is up to you.

Cardinal supports a “mobile first,” device-agnostic approach to responsive design. That is, your application’s design shouldn’t care about the pixel dimension of the device it’s being viewed upon. The content within the design should dictate how and where the layout should change or respond.

The idea is that if you design for the smallest devices (and least capable browsers) first, it forces you to prioritize and determine what are the most important data and actions in your application.

The end result is an application that is focused on the key tasks a user is supposed to complete, without the performance issues, extraneous content, and interface fluff that plagues many applications that take a top-down, “PC first” approach.

Class-name conventions

Cardinal uses simple naming conventions for CSS classes that are human readable and easy to interpret. While inspired by the BEM approach, the naming conventions used in Cardinal are far less complex.

.block_name              {} // Block
.block_name-element_name {} // Element
.modifier_name           {} // Modifier

Use hyphens to separate the name of the block from the name of the element, and use underscores to separate words in long names. Modifiers can stand alone, or be chained to blocks and elements, as depicted below.

.post         {} // Block
.post-title   {} // Element
.post-excerpt {} // Element
.post_last    {} // Specific Modifier
.one_fifth    {} // Generic Modifier

Typography

Instead of trying to maintain a baseline grid, Cardinal leverages modular scale, unit-less line heights, and the power of the REM unit to make it possible to resize the typography and layout of your entire application across different devices.

html { font-size: 100%; } // 16px
@media only screen and (min-width: 480px) { html { font-size: 87.5%;  } } // 14px
@media only screen and (min-width: 960px) { html { font-size: 112.5%; } } // 18px

Root font size

By default, Cardinal’s root font size ranges from 16px to 20px, depending on the viewport width. These breakpoints and font sizes should be changed to suit your application. Altering the root font size by one-pixel, up or down, can completely change how a typeface is rendered. Also, your project may require more or less breakpoints where the root font size should change. Or none. It’s your call.

REMs and line height

After declaring the root font size(s) on the <html> element, Cardinal sets the font-size property on the <body> element to 1rem, and the line-height property to a unit-less value.

body {
    color: #444;
    font-size: 1rem;
    font-family: "Helvetica Neue", sans-serif;
    line-height: 1.5;
}

In other words, 1rem = the root font size(s) on the <html> element. When the root font size changes at different viewport widths, this base rem unit will inherit the new value. As a result, all rem units in your application will resize every time the root font size changes.

Modular scale

A modular scale is then calculated with this tool based on the minimum and maximum root font size, in pixels. The scale also uses the “perfect fourth” ratio found commonly in music.

Cardinal uses the values that are generated for sizing the type and layout (font size, line height, margin, padding, etc.) in the framework. You should calculate your own values and use them as you build your responsive application.

Tables

Tables can be tricky to style across multiple devices. Cardinal includes some simple table styles and classes to get you going.

The <table> element is first given a basic layout with flexible table cells.

2 4652 Goodbye false

Layout

Cardinal has several classes for common layout modules, too, like page wrappers and 100% width containers.

Strips

.strip {}

A strip is useful when you want an element to span the entire width of the viewport.

Wrappers

.wrapper {}

A wrapper provides a max-width for the content within it, and centers the content.

Islands

.island {}

An island is a content block that you want to stand out from its surrounding area.

Wells

.well {}

A well is a content block that you want to appear sunken beneath its surrounding area.

Cardinal’s grid system has 6 columns by default, and uses fluid item widths and fixed gutters. Nesting grid elements is a breeze, and you can change how a grid item behaves with human-readable CSS classes.

Grid item widths have percentage values, and the gutters have rem values. The classes for grid item widths are not grid-specific, on purpose, so that they can be used outside of grids, too.

<div class="grid grid_gutter_half text_center">
    <div class="grid-item one_sixth"></div>
    <div class="grid-item one_sixth"></div>
    <div class="grid-item one_sixth"></div>
    <div class="grid-item one_sixth"></div>
    <div class="grid-item one_sixth"></div>
    <div class="grid-item one_sixth"></div>
</div>

You can also change how a grid behaves at common breakpoints. Instead of device-specific class names like .ipad-one_half or .samsung-two_fifths, Cardinal uses “lap-*” to represent devices you can place in your lap (eg. a tablet or small laptop), and “desk-*” to represent devices that rest primarily on your desk.

By default, the min-width values for the @media queries containing these classes are set to 640px and 960px, respectively. Again, you are encouraged to modify these values based on where your own design breaks or looks best.

Using these classes, you can control the grid item width and grid gutter width.

Buttons

Buttons are common web UI components, and some basic button styles are available with Cardinal. While minimal, there is a large variety of style, shape, and size combinations that you can create.

To begin, simply add the .button class to an element. This provides the base styles that can be further modified by adding more specific classes.

<a href="" class="button">Button text</a>

By chaining CSS classes together, you can quickly style anchor links <a href="">, button tags <button>, and input elements <input type="submit"> to taste.

<a href="" class="button [style] [size] [width] [shape] [state]">
